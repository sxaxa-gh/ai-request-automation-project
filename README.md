# AI Request Automation Project

## Project Overview
This project explored the development of an AI-powered request management solution designed to improve the intake and tracking process for analytics requests within a business environment.

The solution combined AI interaction, workflow automation, and Jira ticket management to streamline the submission and processing of requests received by the Data Analytics team.

## Business Problem
Analytics requests were previously managed through manual communication methods, resulting in inconsistent request submissions, incomplete information, communication delays, and inefficient request tracking.

The project aimed to improve:
- Request standardisation
- Workflow efficiency
- Ticket creation consistency
- Operational visibility
- Communication between business stakeholders and the analytics team

## Solution
An AI-powered “Data Request Scrum Master” agent was developed to collect and validate request information through conversational prompts.

The collected information was then processed through a Power Automate workflow and transformed into structured Jira tickets for tracking and prioritisation.

## Tools Used
- ChatGPT Enterprise
- Microsoft Teams
- Power Automate
- Jira
- Workflow Automation
- JSON Processing

## Data Engineering
The project involved collecting structured request data, validating inputs, transforming request information into a standardised format, and mapping fields into Jira issue structures.

Power Automate workflows were used to process and route data between systems, while JSON parsing supported field extraction and automation logic.

## Business Impact
The project demonstrated how AI and workflow automation can improve operational efficiency by reducing manual request handling, improving data consistency, and supporting more structured analytics request management.

The proposed solution could reduce administrative effort, improve request prioritisation, and support scalability across analytics operations.
